# Documentation

These documents are not merely descriptions of the Three Narrative Editor.

They preserve the observations, principles, questions, and discoveries that have emerged during its development.

Most software documentation explains what was built.

These documents attempt to preserve how the understanding that produced the software evolved.

The project is being guided by observation rather than by predetermined design. Ideas are not retained because they are elegant, familiar, or technically convenient. They are retained only when repeated observation continues to support them. Likewise, ideas that fail to survive observation are discarded, even when they would make perfectly reasonable software abstractions.

For that reason, these documents should not be viewed as a finished design. They are a living record of an ongoing process of discovery.

As the project matures, the software may change. These documents are intended to preserve the observations and principles that continue to survive those changes.

The long-term goal is not merely to build software.

The long-term goal is to uncover a sufficiently faithful representation of relationship structure that the software becomes an instrument through which previously unseen structure can become visible.

These documents exist so that the reasoning behind that pursuit is preserved alongside the code that implements it.
